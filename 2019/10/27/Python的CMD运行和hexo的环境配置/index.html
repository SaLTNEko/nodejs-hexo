Python的CMD运行和hexo的环境配置

2019-10-27 教程

写在最前面：大概就是自己脑子一抽明明在有pycharm的情况下硬耗着想用CMD来运行Py脚本（明明连python2/3版本都不好切换 大概觉得用命令行比较帅（x

步骤：
Win键->设置->搜索系统属性->高级->环境变量->在Path中加入python的路径
注意的是不同的路径间用英文的;分开

然后美滋滋地打开git.bash 输入hexo s -d打算开始更博…发现报错
hexo s -d /c/Users/id/AppData/Roaming/npm/hexo: line 12: node: command not found
查了一下居然是因为node.js未指定 熟悉的配方熟悉的味道…再在Path中加入node.js的路径

然而…又报错hexo s -d bash: hexo: command not found
惨哦…怎么办呢…
P…Path?
没错…就是将hexo文件夹下的node_modules文件夹下的.bin文件的完整路径也加到Path里面

重新打开git.bash 输入hexo s -d 运行成功 啪啪啪啪啪

//顺便一提之后自己发现直接打开python.exe也是命令行欸…所以自己在干啥23333

再再补一个玄学 突然发现自己没有分享按钮 就在theme的_config.yml文件里把shareto设置成true 结果d上去懵了 首页文章都在 但是点击阅读全文都是404
发现是自己没有装hexo-helper-qrcode插件的锅
命令行输入$ npm i -S hexo-helper-qrcode
还是那句话…生命在于不停的玄学和克服玄学…(咸鱼
